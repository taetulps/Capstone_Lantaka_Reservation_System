Lantaka — Client Booking Flow

From browsing to confirmed reservation · Controller → View → Components

1
Browse Accommodations
Client lands on the accommodations listing
GET /accommodations RoomVenueController@index client/room_venue.blade.php <x-client_room_venue_card>
🗂️Fetches all available rooms & venues from DB, renders each as a <x-client_room_venue_card> component
👆Clicking a card navigates to /checkout/{category}/{id}
click card
2
View Room / Venue Detail
Detail page with interactive date picker
GET /checkout/{category}/{id} RoomVenueController@show client/room_venue_viewing.blade.php <x-booking_calendar> booking_calendar.js
📅Controller queries RoomReservation or VenueReservation for booked dates, builds $occupiedDates[] array
🧩<x-booking_calendar> receives $occupiedDates as a prop → injected into window.serverOccupiedDates for JS
⚙️booking_calendar.js renders the calendar grid, blocks occupied dates, and sets hidden inputs check_in & check_out
📋Form collects accommodation_id, type, check_in, check_out, pax → submits GET to /booking/prepare
submit booking form
3
Prepare Booking
Controller decides the next step based on type
GET /booking/prepare RoomVenueController@prepareBooking
🔀Checks $request→type — forks to either direct checkout (room) or food options (venue)
🛏 Room Direct to Checkout
↩️Redirects to GET /checkout with all booking params as query string
⏭️Skips food selection entirely
🏛 Venue Food Options Page
client/food_option.blade.php client_food_option.js
🍽️Fetches Food models grouped by food_category
📆Iterates every date in range using CarbonPeriod, renders a meal table per day
🔄AJAX → FoodController@getFoodsAjax (GET /foods/ajax/list) populates food <select> dropdowns
🔘Toggle per date (food_enabled[date]) and per meal (meal_enabled[date][meal])
📤Submits GET → /checkout with full food_selections[date][meal][category] tree
4
Checkout — Build Cart Items
Controller assembles $processedItems from session + DB
GET /checkout ReservationController@checkout session('pending_bookings') Room / Venue / Food
💾Reads session('pending_bookings'), merges new booking under key {type}_{id}_{check_in}_{check_out}, writes back to session
🔍For each booking: queries Room or Venue for name/price/image; queries Food models by IDs from food_selections
🧮Calculates $accommodationTotal (price × days), $foodTotal (food_price × pax per item, skipping disabled dates/meals)
📦Builds $processedItems[] with totals, raw selections, and food model collection → passes to view
return view
5
My Bookings — Cart UI
The checkout page that displays cart items and the order summary
client/my_bookings.blade.php layouts/client.blade.php partials/client/header.blade.php my_booking.js
🏗️Extends layouts/client.blade.php → includes partials/client/header.blade.php in the layout shell
📦 .cart-items (left panel)
🔁@forelse($processedItems) renders each booking as .cart-item div
📊All data embedded as data-* attributes: name, total, id, type, dates, pax, food JSON, food-selections, food-enabled, meal-enabled
🍽️Venues with food show: foods grouped by category + a date×food table
🖱️Click = toggles .highlighted class → triggers my_booking.js
🧾 .checkout-summary (right panel)
🙈Hidden by default; shown when at least one item is selected
📋#summary-items — accommodation rows with subtotals
🍽️#summary-foods — food breakdown by date/meal/category
💰#summary-grand-total — running sum of all selected items
📤Hidden input #selected-items-input holds JSON payload for form POST
⚙️ my_booking.js — Cart Logic
🗃️In-memory cart{} object — keyed by {type}_{id}_{checkIn}_{checkOut}
👆selectItem(el) — reads data-* attrs, re-parses food JSON, recalculates food total (skipping disabled dates/meals), pushes into cart
🔄renderSummaryItems() — writes accommodation subtotal rows into #summary-items
🍱renderSummaryFoods() — writes date-grouped food entries with meal label, category, pax & subtotal into #summary-foods
💵updateGrandTotal() — sums cart[].total → updates #summary-grand-total, calls render functions
📦updateHiddenSelectedItemsInput() — serializes cart to JSON → stuffs into #selected-items-input
✅Form submit guard — prevents submit if cart is empty, shows alert
POST reservation.store
6
Confirm Reservation
Store records in the database
POST /reservation/store ReservationController@store RoomReservation / VenueReservation / FoodReservation
📥Decodes selected_items JSON from the form POST
💾Creates RoomReservation or VenueReservation records, then FoodReservation entries for each food selection
📧Sends confirmation email via Mail facade
🧹Clears session('pending_bookings') on success
